# Дизайн иерархического выбора SSH-серверов

**Статус:** утверждён пользователем 2026-07-18
**Область:** первичная интерактивная настройка `sshmon`

## Цель

Заменить плоский список SSH-хостов на компактное дерево конфигурационных файлов. Интерфейс должен оставаться управляемым на маленьком терминале, не создавать впечатление зависания и позволять выбирать как весь файл, так и отдельные хосты.

## Архитектура

Решение остаётся на Bubble Tea v1 и использует официальный `bubbles/viewport`. Данные разделяются на два уровня:

1. **Источник SSH-конфигурации** — основной `~/.ssh/config` или конкретный Include-файл.
2. **Хост** — отдельный буквальный алиас `Host` из этого источника.

Парсер сохраняет для каждого хоста идентичность источника. Модель setup-TUI строит из неё упорядоченное дерево, а при завершении преобразует только выбранные хосты в `config.Server`.

### Модель данных

- `config.SSHHost` хранит нормализованный путь источника и порядковый номер объявления внутри него.
- Узел источника идентифицируется путём файла, а узел хоста — парой «путь источника + порядковый номер».
- Отображаемая группа вычисляется отдельно: `main` для корневого файла, basename без `.conf` для Include-файла.
- `config.Server` и YAML не получают служебные идентификаторы источника: наружу записывается только уже существующее поле `Group`.

## Имена групп

- Хосты, объявленные непосредственно в `~/.ssh/config`, получают группу `main`.
- Хосты Include-файла получают имя файла без завершающего `.conf`: `prod.conf` → `prod`.
- Правило одинаково для выбора целого файла и отдельных хостов.
- Путь каталога не входит в имя группы.
- Порядок источников и хостов сохраняется таким, каким его вернул парсер.
- Повторяющиеся алиасы не объединяются: внутренняя идентичность строки состоит из источника и позиции хоста в нём.

## Представление

Все файлы при открытии свёрнуты. Видимые строки рисуются терминальной псевдографикой:

```text
▸ ☐ main                         0/3
▾ ◩ prod                         1/2
  ├─ ☑ prod-web   deploy@10.0.0.10:22
  └─ ☐ prod-db    deploy@10.0.0.11:22
▸ ☑ staging                      4/4
```

- `▸` / `▾` обозначают свёрнутое и раскрытое состояние.
- `☐` — ничего не выбрано, `◩` — выбрана часть хостов, `☑` — выбраны все.
- Строка файла показывает `выбрано/всего`.
- Хост отображает алиас и фактический адрес; длинные значения обрезаются по ширине терминала без переноса строк.
- Пустые источники не показываются.
- Нижняя строка всегда содержит число выбранных хостов и краткую подсказку клавиш.

## Управление

| Клавиши | Действие |
|---|---|
| `↑` / `k`, `↓` / `j` | Перемещение по видимым строкам |
| `Enter` на файле | Переключить свёрнутое/раскрытое состояние |
| `→`, `l` | Раскрыть выбранный файл |
| `←`, `h` | Свернуть файл; на хосте перейти к его файлу |
| `Space` | Переключить хост либо все хосты выбранного файла |
| `a` | Выбрать все хосты; повторное нажатие снять весь выбор |
| `Enter` на хосте | Переключить этот хост |
| `q`, `Esc`, `Ctrl+C` | Отменить настройку без записи конфига |

Подтверждение выполняется отдельной подсказанной клавишей `s` (save), только когда выбран хотя бы один хост. Это устраняет конфликт между `Enter` для раскрытия дерева и завершением настройки.

## Прокрутка и изменение размера

- Высота viewport вычисляется из `tea.WindowSizeMsg` за вычетом заголовка и нижней подсказки.
- В viewport попадает только срез видимых строк; свёрнутые дочерние строки не участвуют в навигации.
- После каждого перемещения, раскрытия, сворачивания и resize модель обеспечивает видимость курсора.
- При resize курсор нормализуется: если прежняя строка исчезла, выбирается её родительский файл либо ближайшая допустимая строка.
- На крайне маленьком терминале остаются видимыми текущая строка и возможность отмены; интерфейс не пытается вывести всё дерево.
- Выбор хранится независимо от раскрытия, поэтому сворачивание ничего не снимает.

## Поток данных

1. `config.ParseSSHConfig` читает основной файл и Include-файлы, сохраняя источник каждого хоста.
2. `setup.Run` группирует хосты по источнику и создаёт свёрнутую модель.
3. Пользователь раскрывает нужные файлы и выбирает целые источники или отдельные хосты.
4. При сохранении модель возвращает выбранные `config.Server` с группой, вычисленной из источника.
5. Существующий first-run поток создаёт или безопасно заполняет YAML, перечитывает его и запускает основной TUI без перезапуска.

## Ошибки и крайние случаи

- Если буквальных SSH-хостов нет, сохраняется существующий fallback на ручной шаблон.
- Нечитаемый корневой SSH config остаётся ошибкой; отсутствующий Include без совпадений не создаёт пустую группу.
- Если Include совпал с файлом, который невозможно прочитать или разобрать, setup завершается с явной ошибкой вместо молчаливого пропуска источника.
- Отмена не создаёт и не изменяет YAML.
- Сохранение без выбора блокируется, а подсказка сообщает причину.
- Несколько Include-файлов с одинаковым basename получают одно отображаемое имя группы, но строки источников остаются отдельными и не теряют идентичность. При записи выбранные хосты получают одинаковый `Server.Group`.
- Очень длинные алиасы, IPv6-адреса, Unicode и сотни хостов не должны менять однострочную геометрию списка.

## Совместимость и не-цели

- Не мигрировать Bubble Tea/Bubbles/Lip Gloss на v2.
- Не добавлять поддержку `ProxyJump`, `ProxyCommand` или полную семантику OpenSSH.
- Не менять основной мониторинговый TUI.
- Не менять формат итогового `config.Server`: служебная идентичность источника существует только в `SSHHost` и setup-модели.
- Не добавлять новую зависимость: `bubbles/viewport` уже входит в используемый стек.

## Проверка

### Модульные тесты парсера

- основной файл назначает группу `main`;
- Include-файл назначает basename без `.conf`;
- порядок источников и хостов сохраняется;
- одинаковые алиасы в разных файлах остаются отдельными элементами.

### Тесты модели Bubble Tea

- все источники свёрнуты по умолчанию;
- раскрытие и сворачивание меняет только видимые строки;
- чекбокс файла проходит состояния `☐ → ☑`, а частичный выбор показывает `◩`;
- выбор целого файла и отдельного хоста возвращает правильные группы;
- выбор сохраняется после collapse/expand;
- курсор не выходит за границы после collapse и resize;
- навигация по большому дереву прокручивает viewport и удерживает курсор видимым;
- `a`, отмена и блокировка пустого сохранения работают независимо от размера терминала.

### Системная проверка

- PTY-сценарий на маленьком терминале с несколькими файлами и десятками хостов: раскрытие, прокрутка, частичный и полный выбор, сохранение и немедленный запуск Overview.
- Проверка YAML: группы `main` и basename Include-файлов записаны корректно.
- Регрессии отсутствующего/пустого конфига и headless-поведения.
- `gofmt -d cmd internal`
- `go build ./...`
- `go vet ./...`
- `go test -race -shuffle=on -count=1 ./...`
- `go build -o sshmon ./cmd/sshmon`
- `git diff --check`
